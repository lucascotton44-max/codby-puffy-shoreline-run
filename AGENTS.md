# AGENTS.md — Cod B’y & Puffy: Shoreline Run

## Project Identity

This project is a premium Atlantic Canada side-scrolling game prototype from Lucas & Liam Legacy Studios.

The tone is grounded, cinematic, restrained, and regionally specific. This is not a generic cartoon game, mascot game, fantasy platformer, or mobile-game reskin.

## Credit Discipline

Treat Codex credits as finite.

Before editing:
- Identify the minimum files likely required.
- Avoid broad repo exploration unless necessary.
- Do not rewrite unrelated files.
- Prefer targeted edits over sweeping improvements.
- Reuse existing code and assets before creating new systems.

Hard no:
- No broad refactors without explicit permission.
- No unrelated cleanup.
- No unnecessary dependencies.
- No replacing approved assets unless requested.
- No “while I was here” changes.

## Core Characters

### Cod B’y

Cod B’y is the heavier playable character.

Locked traits:
- Broad blunt Atlantic cod head
- Heavy fleshy downturned mouth
- One short centered chin barbel
- Dark serious intelligent eyes
- Weathered yellow sou’wester
- Weathered yellow oilskin slicker
- Thick beige knit fisherman sweater
- Rugged dark work pants
- Sturdy brown work boots
- Stoic old-soul maritime presence

Avoid:
- Shark traits
- Eel traits
- Reptile traits
- Frog traits
- Cute mascot styling
- Plush toy styling
- Generic fish-man anatomy
- Childish cartoon drift

Gameplay feel:
- Heavy
- Grounded
- Slower
- Strong silhouette
- Readable at small scale

### Puffy

Puffy is the lighter, quicker companion/playable character.

Locked traits:
- Atlantic puffin identity
- Black and white plumage
- Orange puffin beak
- Orange webbed feet
- Smaller than Cod B’y
- Agile upright silhouette
- Calm but slightly playful energy
- Simple readable clothing/accessory design if used

Avoid:
- Penguin drift
- Tropical bird cues
- Plush toy styling
- Childish mascot proportions
- Over-accessorized fantasy styling

Gameplay feel:
- Light
- Quick
- Agile
- Readable
- Companion-like

## Development Rules

1. Make the smallest safe code change that solves the task.
2. Do not modify unrelated files.
3. Preserve existing working behavior unless explicitly asked to change it.
4. Preserve locked visuals, composition, buildings, characters, and approved art direction.
5. Reuse existing systems before creating new ones.
6. Avoid unnecessary dependencies.
7. Keep code readable and maintainable.
8. Keep performance appropriate for a lightweight 2D browser game.
9. Do not introduce expensive visual effects without explicit instruction.
10. Run build/test/smoke checks after changes when available.

## Visual Direction

Preferred:
- Premium 2D side-scroller readability
- Atlantic Canada shoreline atmosphere
- Weathered docks, stone, water, cliffs, fishing/marina textures
- Cool restrained palette
- Strong silhouettes
- Clear gameplay readability

Avoid:
- Fantasy excess
- Toy-like rendering
- Overly cute mascot design
- Generic mobile-game gloss
- Busy UI
- Unreadable parallax layers
- Heavy filters that obscure gameplay

## Required Report Format After Each Codex Task

After making changes, report:

- Files changed
- What changed
- What stayed locked
- Build/test result
- Remaining risks
